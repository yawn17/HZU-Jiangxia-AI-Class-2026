# 模型选择、欠拟合和过拟合实验报告

## 1. 实验目的
通过多项式拟合来交互地探索模型选择、欠拟合和过拟合的概念。使用不同阶数的多项式模型拟合由三阶多项式生成的数据，观察模型复杂度对训练误差和测试误差的影响。

## 2. 实验方法
### 2.1 数据生成
使用以下三阶多项式来生成训练和测试数据的标签：

$$
y = 5 + 1.2x - 3.4\frac{x^2}{2!} + 5.6\frac{x^3}{3!} + \epsilon \quad \text{where} \quad \epsilon \sim \mathcal{N}(0, 0.1^2)
$$

其中 $\epsilon$ 是均值为0、标准差为0.1的高斯噪声。

### 2.2 模型定义
使用线性模型拟合多项式特征。模型不设置偏置项，因为偏置已在多项式特征中实现（包含常数项1）。

### 2.3 训练与评估
- 使用均方误差（MSE）作为损失函数
- 使用随机梯度下降（SGD）优化器，学习率为0.01
- 训练周期：正常拟合和欠拟合实验为400轮，过拟合实验为1500轮
- 每20轮记录一次训练损失和测试损失

## 3. 代码实现

### 3.1 导入必要的库

```python
import math
import numpy as np
import torch
import torch.nn as nn
from d2l import torch as d2l
```

### 3.2 数据生成

```python
max_degree = 20  # 最大多项式阶数
n_train, n_test = 100, 100  # 训练和测试样本数
true_w = np.zeros(max_degree)  # 真正的多项式系数
true_w[0:4] = np.array([5, 1.2, -3.4, 5.6])  # 真正的多项式系数

features = np.random.normal(size=(n_train + n_test, 1))  # 多项式特征
np.random.shuffle(features)  # 将特征打乱
poly_features = np.power(features, np.arange(max_degree).reshape(1, -1))  # 多项式特征
for i in range(max_degree):
    poly_features[:, i] /= math.gamma(i + 1)  # 使用阶乘进行归一化
labels = np.dot(poly_features, true_w)  # 生成标签
labels += np.random.normal(scale=0.1, size=labels.shape)  # 添加噪声

# 转换为 PyTorch 张量
true_w, features, poly_features, labels = [
    torch.tensor(x, dtype=torch.float32) for x in [true_w, features, poly_features, labels]
]
```

### 3.3 评估损失函数

```python
def evaluate_loss(net, data_iter, loss):
    """评估在指定数据集上模型的损失"""
    metric = d2l.Accumulator(2)  # 损失总和, 样本数量
    with torch.no_grad():
        for X, y in data_iter:
            out = net(X)
            y = y.reshape(out.shape)
            metric.add(loss(out, y), y.numel())
    return metric[0] / metric[1]
```

### 3.4 训练函数

```python
import sys
sys.path.append('..')  # 向上1级到正式班目录
from my_utils import *

def train(train_features, test_features, train_labels, test_labels,
          num_epochs=400):
    loss = nn.MSELoss()
    input_shape = train_features.shape[-1]
    # 不设置偏置，因为我们已经在多项式中实现了它
    net = nn.Sequential(nn.Linear(input_shape, 1, bias=False))
    batch_size = min(10, train_labels.shape[0])
    train_iter = d2l.load_array((train_features, train_labels.reshape(-1,1)),
                                batch_size)
    test_iter = d2l.load_array((test_features, test_labels.reshape(-1,1)),
                               batch_size, is_train=False)
    trainer = torch.optim.SGD(net.parameters(), lr=0.01)
    animator = d2l.Animator(xlabel='epoch', ylabel='loss', yscale='log',
                            xlim=[1, num_epochs], ylim=[1e-3, 1e2],
                            legend=['train', 'test'])
    for epoch in range(num_epochs):
        train_epoch_ch3(net, train_iter, loss, trainer)
        if epoch == 0 or (epoch + 1) % 20 == 0:
            animator.add(epoch + 1, (evaluate_loss(net, train_iter, loss),
                                     evaluate_loss(net, test_iter, loss)))
    print('weight:', net[0].weight.data.numpy())
```

## 4. 实验结果

### 4.1 三阶多项式拟合（正常拟合）

使用与数据生成相同阶数（三阶）的多项式特征进行拟合。

```python
train(poly_features[:n_train, :4], poly_features[n_train:, :4],
      labels[:n_train], labels[n_train:])
```

**结果分析：**
- 学习到的权重接近真实权重 [5, 1.2, -3.4, 5.6]
- 训练损失和测试损失都收敛到较低水平（约 0.01）
- 模型能够很好地拟合数据，没有出现欠拟合或过拟合

### 4.2 线性函数拟合（欠拟合）

仅使用一阶多项式特征（即线性特征）进行拟合。

```python
# 从多项式特征中选择前2个维度，即1和x
train(poly_features[:n_train, :2], poly_features[n_train:, :2],
      labels[:n_train], labels[n_train:])
```

**输出结果：**

**结果分析：**
- 学习到的权重与真实权重差异较大
- 训练损失和测试损失都较高（约 1.0）
- 模型过于简单，无法捕捉数据的非线性特征，出现欠拟合

- 模型过于复杂，学习了数据中的噪声，出现明显的过拟合现象

## 5. 分析与讨论

### 5.1 损失曲线对比
- **正常拟合**：训练损失和测试损失同步下降并收敛到相近的低值
- **欠拟合**：训练损失和测试损失都较高，模型无法充分学习数据特征
- **过拟合**：训练损失极低，但测试损失明显高于训练损失，存在明显的泛化差距

### 5.2 模型复杂度的影响
1. **模型过于简单（欠拟合）**：无法捕捉数据的真实规律，表现为高偏差
2. **模型适当复杂（正常拟合）**：能够很好地拟合数据同时保持良好的泛化能力
3. **模型过于复杂（过拟合）**：过度拟合训练数据中的噪声，表现为高方差


## 6. 结论

本实验通过多项式拟合的实例，直观展示了模型选择、欠拟合和过拟合的概念：
1. 当模型复杂度与数据真实复杂度匹配时，模型表现最佳（正常拟合）
2. 当模型过于简单时，会出现欠拟合现象，表现为高偏差
3. 当模型过于复杂时，会出现过拟合现象，表现为高方差

实验结果验证了机器学习中偏差-方差权衡的基本原理，为实际模型选择提供了重要参考。
